# Бот мониторинга комментариев ВК

Бот принимает ссылку на пост в личке сообщества, сохраняет её в `SQLite` и регулярно проверяет комментарии. Если появляется новый комментарий, бот отправляет уведомление в ту же личку.

## Что умеет

- принимает ссылку на пост вида `https://vk.com/wall-123_456`;
- сохраняет пост в мониторинг;
- присылает уведомление о новом комментарии;
- показывает список отслеживаемых постов;
- удаляет один пост из мониторинга;
- отключает весь мониторинг для пользователя.
- пускает к управлению только после секретной команды.

## Команды в личке боту

- просто отправьте ссылку на пост;
- сначала один раз отправьте `доступ <секретный_код>`;
- `список` или `list` — показать активные посты;
- `удалить 2` — удалить пост по номеру из списка;
- `удалить https://vk.com/wall-123_456` — удалить по ссылке;
- `стоп` — убрать все ваши посты из мониторинга;
- `помощь` или `help` — показать подсказку.

## Что нужно подготовить

1. Сообщество ВК, от имени которого бот будет писать вам в личку.
2. Включённые сообщения сообщества.
3. Токен сообщества для сообщений.
4. Желательно отдельный токен для чтения комментариев:
   `VK_READER_TOKEN`.

Если `VK_READER_TOKEN` не указан, бот пробует читать комментарии токеном сообщества. Для части открытых постов этого достаточно, но для чужих постов стабильнее иметь отдельный токен чтения.

## Установка

1. Откройте терминал.
2. Перейдите в папку проекта:

```bash
cd "/Users/darya_botova/Documents/Подборки/vk_comment_monitor"
```

3. Создайте виртуальное окружение:

```bash
python3 -m venv .venv
```

4. Активируйте его:

```bash
source .venv/bin/activate
```

5. Установите зависимости:

```bash
pip install -r requirements.txt
```

6. Скопируйте пример настроек:

```bash
cp .env.example .env
```

7. Откройте `.env` и заполните:

- `VK_GROUP_ID` — ID сообщества;
- `VK_GROUP_TOKEN` — токен сообщества;
- `VK_READER_TOKEN` — токен для чтения комментариев, если есть;
- `VK_READER_TOKEN_TTL_SECONDS` — срок жизни `VK_READER_TOKEN` в секундах (по умолчанию `86400`);
- `VK_READER_TOKEN_WARN_BEFORE_SECONDS` — за сколько секунд предупредить в личке до истечения (по умолчанию `10800`);
- `ALLOWED_USER_ID` — ID вашей личной страницы, только этот пользователь сможет управлять ботом и получать уведомления;
- `STRICT_DIALOG_MODE` — `1` (по умолчанию) включает жёсткую привязку к одному диалогу;
- `ACCESS_CODE` — секретный код для доступа к боту;
- `CHECK_INTERVAL_SECONDS` — частота проверки, например `90`.

8. Запустите бота:

```bash
python3 bot.py
```

## Настройка сообщества ВК

1. Включите сообщения сообщества.
2. Убедитесь, что вы уже написали этому сообществу хотя бы одно сообщение.

Это важно: без открытого диалога сообщество не сможет отправлять вам личные сообщения.

Long Poll для работы этого MVP не нужен. Бот сам периодически проверяет входящие сообщения сообщества через API.

## Первый запуск

1. Отправьте боту команду `доступ <секретный_код>`.
2. После подтверждения отправьте ссылку на нужный пост.
3. Бот ответит, что пост добавлен в мониторинг.
4. Когда появится новый комментарий, бот пришлёт уведомление.

## Как работает первое добавление

При первом добавлении бот запоминает уже существующие комментарии и не шлёт на них старые уведомления. Он уведомляет только о тех комментариях, которые появились после добавления поста в мониторинг.

## Ограничения MVP

- бот отслеживает верхнеуровневые комментарии к посту;
- если за один цикл появится очень много новых комментариев, бот обрабатывает их пачками;
- если пост или комментарии стали недоступны, бот пришлёт сообщение об ошибке доступа только в лог терминала.

## Деплой на сервер

Для постоянной работы 24/7 используйте отдельный сервер. Готовая инструкция находится в [DEPLOY.md](DEPLOY.md), а пример `systemd`-сервиса лежит в `deploy/vk-comment-monitor.service`.

## Railway

Для этого бота из облачных платформ удобнее всего Railway:

- поддерживает GitHub-деплой;
- поддерживает Python;
- позволяет задать свой `Start Command` или использовать `Dockerfile`;
- поддерживает Volumes для хранения `SQLite`.

Если будете разворачивать на Railway, для `DATABASE_PATH` лучше использовать путь внутри volume, например:

```env
DATABASE_PATH=/data/monitor.sqlite3
```

И затем примонтировать volume в `/data`.
